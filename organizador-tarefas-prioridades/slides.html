Prioriza

Um organizador de tarefas que calcula o que fazer agora por urgência × importância — do Setup 0 ao app pronto.

Tema escolhido: Organizador de Tarefas e Prioridades · Inspiração visual: Jenkins

O problema

Uma lista não diz o que fazer primeiro

Todo mundo tem uma lista de tarefas. O difícil é decidir a ordem: o que atacar agora, o que agendar, o que passar adiante e o que simplesmente cortar.

O Prioriza transforma a lista solta numa recomendação: "comece por esta".

O conceito (não é achismo)

Matriz de Eisenhower + GTD + MoSCoW

  • Eisenhower (urgente × importante): o motor da priorização — Covey formalizou os 4 quadrantes.
  • GTD (David Allen): capturar tudo fora da cabeça.
  • MoSCoW (Dai Clegg): classificar o essencial.
Faça agora Agende Delegue Elimine

Fontes reais e linkadas — registradas no GLOSSARIO.md antes de qualquer código.

As regras (Setup 0)

Como a IA respondeu em TODAS as etapas

  • Partes pequenas, com testes junto (positivos e negativos).
  • Prioridade calculada por função pura, nunca digitada.
  • Acessibilidade WCAG AA com relatório de contraste ao vivo.
  • Usabilidade sem becos, LGPD com dados fictícios, tudo documentado.
  • Decisões do usuário se perguntam — ex.: a marca de inspiração.

Arquitetura

Regra pura × mundo externo

  • logica.js — funções puras (priorização, status, filtros, contraste). Testável e previsível.
  • repositorio.js — localStorage, id e data (o não-puro).
  • app.js — só liga a tela às regras.

Tempo e aleatoriedade entram "de fora" — as funções continuam puras.

Qualidade de código

Lint, formatação e hook de commit

  • ESLint (npm run lint) aponta erros e maus hábitos no JavaScript.
  • Prettier (npm run format) mantém tudo num padrão único.
  • Husky roda os dois no pre-commit, em qualquer branch.

Código com erro de lint ou fora do padrão não entra no histórico; os .gitignore/.eslintignore/.prettierignore mantêm fora o que não é fonte.

A jornada

Prompt 0 → 12, um objetivo por vez

  • 0–1 Setup + fundamentação (fontes, glossário).
  • 2–4 Lógica pura → primeira tela → persistência.
  • 5–7 Lista por quadrante → ciclo de status → painel "Foco do dia".
  • 8–10 Filtros persistidos → ações destrutivas → exportar/limpar (LGPD).
  • 11–12 Acessibilidade + E2E → README + estes slides.
Acessibilidade

Cada par de cores é medido por funções puras e mostrado ao vivo no rodapé. Um teste "guarda" reprova qualquer cor abaixo do AA.

Inspiração visual

Jenkins, o mordomo do CI/CD

Paleta oficial: Worn Navy, Medium Carmine (a gravata), Bismark. O vermelho puro do Todoist reprovava no AA; o carmim do Jenkins passou (4.79:1) — mantido autêntico.

A estética nunca fura a acessibilidade: quando a cor de marca reprova, ajusta-se o tom.
